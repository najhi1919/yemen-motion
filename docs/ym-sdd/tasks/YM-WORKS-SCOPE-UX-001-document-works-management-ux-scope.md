# YM-WORKS-SCOPE-UX-001 — Works Management UX Scope

## Scope

يوثق هذا القرار النطاق الوظيفي وتجربة المستخدم والمعمارية الداخلية المعتمدة للتنفيذ القادم لقسم إدارة الأعمال، دون تنفيذ Backend أو Frontend في هذه الخطوة.

## Main route

- الرابط الرئيسي: `/admin/works`.
- يمثل نقطة الدخول إلى إدارة الأعمال والنظرة العامة.

## Sidebar behavior

- لا تظهر الأقسام الفرعية دائمًا في القائمة الجانبية العامة.
- تظهر فقط داخل `/admin/works` أو أي مسار متفرع منه.
- تعرض كتوسعة مرئية تحت رابط "الأعمال" داخل Sidebar الحالي، وليست روابط عامة مستقلة ودائمة.

## Internal sections

| المسار | القسم |
|--------|-------|
| `/admin/works` | النظرة العامة |
| `/admin/works/all` | كل الأعمال |
| `/admin/works/review` | طلبات المراجعة |
| `/admin/works/visibility` | الظهور والتمييز |
| `/admin/works/reports` | البلاغات والمخالفات |
| `/admin/works/taxonomy` | التصنيفات والوسوم |
| `/admin/works/activity` | سجل الأعمال |
| `/admin/works/settings` | إعدادات وصلاحيات الأعمال |

## Section responsibilities

### النظرة العامة

- إحصائيات الأعمال وملخص قيد المراجعة.
- ملخص المنشور والمخفي والمميز والبلاغات.
- آخر الإجراءات.

### كل الأعمال

- قائمة جميع الأعمال مع البحث والفلاتر والفرز.
- التفاصيل والمعاينة والملفات والإجراءات الفردية والجماعية.
- التصنيفات والوسوم وربط العمل بالمصمم.
- أسباب الرفض وطلبات التعديل وسجل مصغر داخل التفاصيل.

### طلبات المراجعة

- حالات `submitted` و`in_review` و`changes_requested`.
- فتح العمل وبدء المراجعة والاعتماد وطلب التعديل والرفض مع سبب.
- النشر بعد الاعتماد عند سماح الصلاحية، مع سجل للمراجعة.

### الظهور والتمييز

- المنشور والمخفي والمميز والمثبت.
- النشر والإخفاء والتمييز وإزالته والتثبيت وإلغاؤه.
- إدارة الظهور العام.

### البلاغات والمخالفات

- الأعمال المبلّغ عنها وعدد البلاغات وأنواعها وحالاتها.
- مراجعة البلاغ، وإخفاء العمل أو رفضه أو طلب تعديله، وتجاهل البلاغ.
- استعادة العمل وأرشفة البلاغ وسجل البلاغات.

### التصنيفات والوسوم

- إدارة التصنيفات والوسوم وعدد الأعمال داخل كل تصنيف.
- الإضافة والتعديل والتعطيل.
- نقل الأعمال إلى تصنيف وتحديث وسوم أعمال محددة.

### سجل الأعمال

- الأحداث المعتمدة: `work.created`, و`work.submitted`, و`work.review.started`, و`work.approved`, و`work.changes_requested`, و`work.rejected`, و`work.published`, و`work.hidden`, و`work.featured`, و`work.unfeatured`, و`work.archived`, و`work.restored`, و`work.category.changed`, و`work.tags.updated`, و`work.report.reviewed`.

### إعدادات وصلاحيات الأعمال

- مصفوفة صلاحيات تغطي العرض والإنشاء والتعديل والمراجعة والاعتماد والرفض وطلب التعديل.
- تغطي النشر والإخفاء والتمييز والأرشفة والاستعادة والإجراءات الجماعية.
- تغطي إدارة التصنيفات والوسوم والبلاغات، وعرض السجل، وإدارة إعدادات الأعمال.

## Global UX rules

- تخصيص حقل البحث العلوي لسياق كل صفحة.
- توفير فلاتر كاملة ومفهومة وtooltips واضحة.
- دعم الترتيب التصاعدي والتنازلي للجداول.
- ربط الإحصائيات بالبطاقات والرسوم.
- تكبير خط البيانات وتحسين الوضوح.
- تمييز الأقسام والحالات بصريًا بالألوان.
- منع `v-html` وconsole logs ومكتبات Charts الخارجية.
- عدم إغلاق أي قسم قبل اكتماله وظيفيًا حسب نطاقه المعتمد.

## Permission model direction

- تبنى الصلاحيات على مستوى العمليات والأزرار والحقول الحساسة، لا على مستوى الصفحة فقط.
- تشمل من يستطيع العرض والإنشاء والتعديل والمراجعة والاعتماد والرفض وطلب التعديل.
- تشمل النشر والإخفاء والتمييز والأرشفة والاستعادة والإجراءات الجماعية.
- تشمل إدارة التصنيفات والوسوم والبلاغات، وعرض سجل الأعمال، وإدارة إعدادات الأعمال.

## Audit direction

- يسجل Audit آمن لكل إجراء مهم في دورة حياة العمل والمراجعة والظهور والتصنيف والبلاغات.
- تعتمد أسماء أحداث سجل الأعمال المحددة في هذا القرار عند التنفيذ.
- يمنع اعتبار قائمة الأحداث توثيقًا لتنفيذها الحالي؛ هي عقد تسمية واتجاه للتنفيذ القادم.

## Visual direction

- يحافظ القسم على Sidebar الحالي ويعرض التفرعات سياقيًا تحت رابط "الأعمال".
- تستخدم حالات وألوان واضحة، وخط بيانات أكبر، وبطاقات ورسوم مرتبطة بالإحصائيات.
- تنفذ الرسوم دون مكتبات Charts خارجية، مع tooltips واضحة وتوافق مناسب مع RTL.

## Deferred items

- لا يوجد تنفيذ برمجي في هذه الخطوة.
- لا توجد مراجعة بصرية شاملة الآن.
- لا تبنى Orders أو Contests أو Bookings أو Finance ضمن هذا القسم.
- تبقى هذه الأقسام مستقلة، ويمكنها الاستفادة من Works لاحقًا.
